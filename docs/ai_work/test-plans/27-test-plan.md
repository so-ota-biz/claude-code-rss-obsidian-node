# Test Plan

## Ticket

GitHub Issue #27 - 503エラーの解消
https://github.com/so-ota-biz/ai-agent-orchestration-settings/issues/27

## Test Targets

- `src/lib/rss.ts`の`fetchPostsForAccount`関数のエラーハンドリング
- HTTPエラー発生時のログメッセージ出力
- 複数アカウント処理時のエラー波及防止
- エラー時の戻り値が後続処理に影響しないこと

## Test Cases

### 単体テスト

1. **正常系**
   - RSSHub APIが正常にレスポンスを返す場合
   - 投稿データが適切にパースされることを確認

2. **503エラー（Twitter API未設定）**
   - rss-parserが503エラーをthrowする場合
   - エラーメッセージに「Twitter API設定」への言及があることを確認
   - 空配列が返されることを確認

3. **その他のHTTPエラー**
   - 404エラー（存在しないアカウント）
   - 429エラー（レート制限）
   - 500エラー（サーバーエラー）
   - 適切なエラーメッセージが出力されることを確認

4. **ネットワークエラー**
   - タイムアウト
   - 接続失敗
   - 適切なエラーメッセージが出力されることを確認

### 統合テスト

1. **複数アカウント処理**
   - 一部のアカウントで503エラーが発生
   - 他のアカウントの処理が継続されることを確認
   - エラーが発生したアカウントは空データとして処理されることを確認

2. **メインプロセス継続確認**
   - すべてのアカウントで503エラーが発生
   - プログラムがfatalエラーで終了しないことを確認
   - 空のダイジェストファイルが生成されることを確認

### 手動テスト

1. **実際のRSSHub環境でのテスト**
   - Twitter API未設定でRSSHubを起動
   - `npm run run`実行
   - 改善後のエラーメッセージが表示されることを確認

2. **手順書検証**
   - 作成したトラブルシューティング手順書に従って設定
   - 503エラーが解消されることを確認

## Risks / Unknowns

- rss-parserライブラリの内部エラーハンドリング仕様が不明確
- RSSHubのHTTPエラーレスポンス形式が変更される可能性
- Twitter API設定手順が変更される頻度
- モックテスト時にrss-parserの実際の例外形式を正確に再現できるか不明
