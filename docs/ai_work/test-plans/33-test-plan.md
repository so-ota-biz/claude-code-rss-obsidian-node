# Test Plan

## Ticket

https://github.com/so-ota-biz/ai-agent-orchestration-settings/issues/33

## Test Targets

- VPS 環境での Docker Compose 動作確認
- RSSHub + Redis サービスの正常稼働
- Node.js バッチ処理の実行とデータ出力
- 環境変数とコンフィギュレーション設定
- データ永続化とバックアップ機能
- cron ジョブによる自動実行
- ネットワーク接続と API アクセス
- セキュリティ設定の有効性

## Test Cases

### VPS環境構築テスト
1. **VPS インスタンス作成・初期設定**
   - OS インストール（Ubuntu 22.04 LTS 推奨）
   - Docker / Docker Compose インストール
   - 必要なポート開放（1200, SSH）
   - ファイアウォール設定
   - SSH 公開鍵認証設定

2. **アプリケーション配置テスト**
   - Git リポジトリクローン
   - 環境変数ファイル（.env）配置
   - docker-compose.yml ファイル確認
   - 必要なディレクトリ作成

### 動作確認テスト
3. **RSSHub サービステスト**
   - `docker compose up -d` による起動
   - http://[VPS_IP]:1200 でのアクセス確認
   - Redis コンテナの正常動作確認
   - ヘルスチェック API の応答確認

4. **Node.js バッチ処理テスト**
   - 依存関係インストール（`npm install`）
   - TypeScript コンパイル確認（`npm run check`）
   - 手動実行による動作確認（`npm run run`）
   - RSS データ取得の成功確認
   - Gemini API 呼び出し成功確認
   - 出力ファイル生成確認

### データ永続化テスト
5. **永続化ボリュームテスト**
   - Redis データの永続化確認
   - コンテナ再起動後のデータ保持確認
   - バックアップ作成とリストア手順の実行

6. **Dropbox ストレージテスト**（Dropbox 使用時）
   - OAuth 認証設定確認
   - ファイルアップロード成功確認
   - トークン自動更新機能確認

### 自動化テスト
7. **cron 設定テスト**
   - crontab 設定とスケジュール確認
   - 自動実行の動作確認（短間隔でのテスト実行）
   - ログファイル出力確認

### パフォーマンステスト
8. **負荷・スループットテスト**
   - 複数アカウント同時処理時の負荷確認
   - メモリ使用量の監視
   - CPU 使用率の確認
   - ネットワーク帯域幅の使用量確認

### セキュリティテスト
9. **セキュリティ設定確認**
   - SSH 接続のセキュリティ設定
   - ファイアウォール設定の有効性確認
   - 環境変数の適切な保護確認
   - Docker セキュリティ設定確認

### 障害テスト
10. **障害対応テスト**
    - ネットワーク障害時の挙動確認
    - API レート制限時の処理確認
    - ディスク容量不足時の対応確認
    - OOM（Out of Memory）発生時の処理

## Risks / Unknowns

- VPS スペックの実際の十分性（実稼働まで不明）
- 地理的な場所によるAPI レスポンス時間の変化
- VPS プロバイダー固有の制限やポリシーの影響
- Docker のストレージドライバーやネットワーク設定による性能差
- 長期運用時の安定性（メモリリーク、リソース枯渇等）
- バックアップ・リストア手順の実効性（実際の障害時まで未知）
